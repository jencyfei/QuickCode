## 💡 设计理念

### 1. 信息层级
**主要信息**：标签名称 + 短信数量  
**次要信息**：图标、颜色

**原则**：一眼看到最重要的内容

---

### 2. 空间利用
**水平布局**：左右分布，符合阅读习惯  
**垂直压缩**：减少高度，提高信息密度

---

### 3. 交互简化
**点击整卡片**：查看该标签的短信  
**需要编辑**：点击右上角"管理"按钮

**原则**：主要操作最简单，次要操作不干扰

## 🔧 技术细节

### 数字样式设计

```css
.tag-count {
  font-size: 18px;        /* 大字号，醒目 */
  font-weight: 600;       /* 半粗体，平衡 */
  color: #FF6B9D;         /* 粉色，与主题一致 */
  min-width: 30px;        /* 最小宽度，对齐 */
  text-align: right;      /* 右对齐，整齐 */
}
```

**为什么选择这些值**：
- **18px**：比标签名（16px）略大，突出重要性
- **600**：比正常（400）粗，比加粗（700）轻，视觉平衡
- **粉色**：与主题色一致，形成视觉联系
- **右对齐**：数字右对齐更易于比较大小

├──────────────────────┤
│ [🏦] 银行         12  │

```

```
